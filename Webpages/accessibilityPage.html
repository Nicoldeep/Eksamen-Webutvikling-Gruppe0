ACCESSIBILITY PAGE



Welcome-Page

The way we designed the welcome page we did so with the intend to make it easy to navigate with the header and buttons. General info about the restaurant in the footer.
We deliberatly chose to use these colors to make it easy on the eyes and readable with text contrasting the background. And making the background colors match to make it visually pleasing.
Menu

In our menu, accessibility and universal design are factors that have been taken care off, because we believe that everyone should be taken into account! Good contrasts in all of our text will help users with impaired vision. Ingredients in the food are one of the most important things for users to know, in our menu the allergens are informed with emojis/Figure, so that you can know about allergens even if you cannot read. By hovering over, the allergens come up as text as well. All images have an alternative text. The site has access to zoom functions to read text better and in general the menu is very simple and has a clear layout so that you can easily see the dish, the price, and you can get valuable information about the content and allergens. "WAVE web accessibility tool" helped ensuring this.
Booking-Page

In the booking page we used appropriate headers to convey information in a structured manner, in addition to bold text to highlight the most important info, which could be helpful for those with visual impairments. Propper form labels were used in case the user uses a screen reader and needs additional information on what to write in the form. For a bit of contast in the page, we used a vibrant background for the form, so it's easier to find where to book a table. Scalability is important for us, so that the website doesn't lose any integrity when accessed via phone, because some may not have access to a computer. In case the user has problems loading in the header photo or website logo, we do provide an alternative text.
Topical-Page

This page was designed with the focus off mobile screen use, and then overrided elements with different breakpoints so it could expand to tablets and desktops too and still look good. We wanted to make it possible to get clear information regardless the device used. Implementing videoes for the topics to make it more user friendly. (The information are just different topics we have written about, it doesnt have a flow.)
